Nmap Port Scanning Overview

Nmap supports various port scanning methods to discover open ports and services on target machines. Here's a summary of key scanning methods:

  • TCP SYN (Stealth) Scan (-sS): Fastest scan for popular TCP ports. Stealthier than connect scan, works on all TCP stacks.
  • TCP Connect Scan (-sT): Uses system call to scan machines. Preferred for unprivileged Unix users and IPv6 targets.
  • UDP Scan (-sU): Scans UDP ports to find security vulnerabilities.
  • TCP FIN, NULL, and Xmas Scans (-sF, -sN, -sX): Special purpose scans for bypassing firewalls. Some systems, especially Windows, may not exhibit expected behavior.
  • TCP ACK Scan (-sA): Maps out firewall rulesets. Cannot distinguish open from closed ports.
  • TCP Window Scan (-sW): Similar to ACK scan, detects open vs. closed ports on certain machines.
  • TCP Maimon Scan (-sM): Firewall-evading scan, similar to FIN scan with ACK flag. Works against fewer systems than FIN scan.
  • TCP Idle Scan (-sI <zombie host>): Stealthiest scan exploiting trusted IP address relationships. Slow and complex.
  • IP Protocol Scan (-sO): Determines supported IP protocols (TCP, ICMP, IGMP, etc.).
  • TCP FTP Bounce Scan (-b <FTP bounce proxy>): Deprecated method tricking FTP servers for port scans.

Port Selection Examples: Specify ports with -p option using numbers, names, ranges, or protocols.

Timing-related Options: Control scan times with options like -T0 through -T5, --min-rtt-timeout, --max-rtt-timeout, etc.

Output Format and Verbosity Options: Choose output format with -oN, -oG, -oX. Increase verbosity with -v or debugging with -d.

Firewall and IDS Evasion Options: Numerous options for evading firewalls and IDSs.

Specifying Targets: Specify target hosts using names, IP addresses, files, or even randomly generated ones.

Miscellaneous Options: -6 for IPv6 scanning, -r for randomizing port scan order, -Pn to skip ping test.
